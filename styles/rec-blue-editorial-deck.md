# rec-blue-editorial-deck

别名：蓝色 REC 编辑感演示风格、REC 蓝色编辑风、黑白蓝现代演示风、录制感视觉 PPT。

## 风格定位

生成具有独立设计杂志感、录像界面感和强烈黑白蓝视觉记忆的图片版 PPT、演示封面、章节页、方法论页、产品概念页和文章配图。参考来源中的招聘内容只作为视觉样本，不作为默认主题；后续使用时应服务具体 PPT 内容，而不是复刻招聘海报。

核心气质：冷静、先锋、编辑化、强留白、强排版、轻微实验性。适合创意技术、AI 工具、产品方法论、设计系统、个人品牌演示、工作坊讲义、视觉提案和需要鲜明记忆点的观点型演示。

## 视觉 DNA

- 冷白灰纸张底色，带轻微噪点、扫描、半调或低保真印刷质感。
- 黑白摄影或灰阶实物图作为主要视觉材料，人物、设备、空间、文档、手部动作、工作场景都可以使用。
- 高饱和钴蓝作为唯一强强调色，用于横向信息条、几何符号、局部物体重着色、关键词或少量 UI 标记。
- 巨大斜体无衬线标题是画面的主视觉锚点，可以跨越图片区和文字区，但必须保持可读。
- 小号 UI 式标记形成“正在录制 / 设计档案 / 编辑部页面”的气质，例如 `REC`、黑点、短斜杠、编号、竖排网址、侧边标签、极小页脚。
- 直角、硬边、无圆角。色块、图片裁切、信息条和版面边界都应是干净的矩形或锐利几何。
- 画面允许局部粗糙、颗粒、错位和印刷瑕疵，但整体网格、对齐和标题层级必须清楚。

## Style Lock

用于多页图片版 PPT 的跨页控制系统：

- 字体气质：现代无衬线，接近 Helvetica Neue、Neue Haas Grotesk、Arial、Inter、SF Pro、思源黑体、阿里巴巴普惠体；主标题优先使用大号斜体或倾斜处理，正文使用轻量或常规字重。不要使用手写体、复古衬线、可爱圆体或装饰字体。
- 语言规则：默认中文优先，尤其标题、章节名、模块名、结论句和输出文档正文必须优先中文；英文只保留产品名、模型名、品牌名、代码/API 术语和必要专有名词，例如 Claude、Image2、API、XML 标签名。极少量英文可作为 UI metadata、网址占位或短标签，但不能喧宾夺主。
- 字号层级：H1 是画面最大元素，占画面宽度 35%-70%，可倾斜、压图、跨区；H2 约为 H1 的 30%-45%；模块标题约为 H1 的 16%-24%；正文和标签必须短句化，避免长段落。
- 版式网格：默认 16:9 使用 72-120px 安全边距；封面可以使用非对称大标题 + 大留白 + 黑白摄影锚点；内页使用清晰网格、大片留白、1 个主标题区、1 个视觉锚点和 2-3 个信息点。元素可偏左、偏上或偏下，但不得随机漂浮。
- 页面角色：封面页和章节页可以更像视觉海报，使用巨大标题、黑白摄影和钴蓝符号。普通内页必须更克制，信息优先，摄影和纹理退到背景或边缘，蓝色只用于强调一个关键句、关键词或结构节点。
- 装饰元素：可重复使用黑点 `REC`、双斜杠编号、竖排小字、蓝色横条、蓝色八角星/星号/爆炸形、半调噪点、扫描线、错印纹理、极小页脚和直角裁切。每页只选择 2-4 种，保持跨页一致。
- 风格隔离：只使用冷白黑白蓝编辑排版语言，不混入深色 HUD、金融咨询网格、奶油商业模板、渐变科技风、3D 物体、卡通插画、软圆角卡片、玻璃拟态、彩色贴纸或其它历史参考风格。不要把参考图里的招聘语义、工作岗位、特定网站或工作室标志带入后续 PPT，除非用户明确要求。
- 色彩比例：65%-78% 冷白/浅灰背景，12%-25% 黑白摄影或灰阶纹理，8%-12% 近黑文字和结构，3%-6% 钴蓝强调。蓝色必须少而强，不要扩展成多色蓝紫渐变。
- 色号：背景 `#F1F1EE` / `#EDEDE9` / `#F7F7F3`，主文字 `#060606` / `#111111`，弱文字 `#4B4B4B` / `#6F6F6A`，主蓝 `#005BFF` / `#004CFF` / `#0066FF`，浅灰边界 `#D6D6D0`，噪点黑灰 `#1A1A1A`，局部重色蓝可使用 `#0057FF`。
- 边框与容器：统一使用直角矩形、硬边图片裁切、无阴影或极弱投影。内部可用 1px 浅灰线、黑色细线、蓝色实心条和无圆角信息块。禁止全页黑色外框；内部线条必须离开画布边缘或被图片/留白打断。
- 页眉页脚行为：可在顶部或角落放极小 `//01`、`REC`、黑点、章节短码、竖排标签或项目名。不要默认添加日期、时间、生成时间、导出时间或当前日期。页眉页脚必须像界面标记，不像品牌横幅。
- Logo/字标行为：默认不让 Image2 生成 logo、品牌字标、水印或可复用品牌标识。如果需要品牌标识，应预留安静位置，后期作为固定资产叠加；不要要求模型跨页复刻同一个 logo。
- 文本密度：默认低密度。封面 1 个巨大标题 + 1 个短副标题 + 1-2 个小标签。普通内页 1 个标题 + 1 个短结论或副标题 + 最多 2-3 个短信息点，每点 1 行。报告型页面也要先压缩成关键词、数字或短句。
- 禁用项：不要招聘海报语义漂移，不要圆角卡片，不要渐变背景，不要彩色插画，不要 3D mockup，不要柔和阴影，不要紫蓝科技模板，不要密集小字，不要整页黑框，不要随机英文假字，不要在未确认日期时写日期。

## 色彩建议

- 冷白背景：`#F1F1EE`, `#EDEDE9`, `#F7F7F3`
- 主文字：`#060606`, `#111111`
- 辅助文字：`#4B4B4B`, `#6F6F6A`
- 主强调蓝：`#005BFF`, `#004CFF`, `#0066FF`
- 边界浅灰：`#D6D6D0`, `#C9C9C3`
- 噪点/半调：`#1A1A1A` 低透明度
- 色彩使用：蓝色只承担一个主要视觉功能，例如信息条、关键词、符号、局部物体重着色或路径高亮；同一页不同时大面积使用多个蓝色系统。

## PPT 页面适配

- 封面页：巨大斜体标题 + 黑白摄影锚点 + 钴蓝符号或横条 + 极小 `REC`/编号/项目标签。
- 目录页：大留白 + 3 到 5 个章节标题，使用 `//01`、`//02`、黑点或蓝色短条作为章节标记。
- 核心观点页：一条巨大判断句占主导，蓝色高亮关键词，右侧或底部放黑白低对比视觉。
- 框架页：2-3 个直角信息区，少量蓝色连接线或蓝色节点，不做复杂图表。
- 流程页：横向蓝色条或细线贯穿，3-5 个步骤，搭配极小编号和黑白材料图。
- 对比页：左右或三栏硬切分，黑白与蓝色形成对照；每栏只保留短标题和 1 行解释。
- 数据页：只突出 1-3 个大数字，使用蓝色作为关键数字、进度条或局部标记，不做密集仪表盘。
- 结论页：大号斜体结论 + 极少行动点 + 安静黑白背景或局部蓝色符号。
- 结束页：极简，保留巨大关键词、一个蓝色符号和小号联系/下一步信息，不使用默认商务“Thank You”模板。

PPTX 交付时默认把整页图片铺满幻灯片，不要求文字、数字、图表、标签可编辑。优先用 Image2 直接生成完整页面视觉。

## 图片构图规则

1. 先确定一个视觉锚点：巨大斜体标题、黑白摄影、蓝色符号、蓝色横条或关键数字。
2. 标题必须一眼可读，可以压在图片上，但不能被人物、纹理或符号破坏识别。
3. 蓝色只放在最需要被记住的位置；不要让蓝色平均分布。
4. 摄影素材优先灰阶化、颗粒化、半调化，局部可被钴蓝重着色。
5. 页面边缘可以有微小 UI 标记，但不要形成外框。
6. 使用强烈留白，留白本身是风格的一部分。
7. 内页不要每页都做成封面海报；普通内容页要降低摄影和纹理的干扰。
8. 所有块面和图片裁切保持直角硬边。

## 默认输出参数

- 默认比例：16:9。
- 默认质量：Image2，高分辨率，清晰中文，锐利边缘，纸张噪点和半调纹理自然。
- 默认输出：图片序列；用户说 PPT 时组装为图片版 PPTX。
- 默认语言：中国用户场景下中文优先，必要英文作为小号 UI 标记或术语。
- 默认密度：低信息密度，适合演示现场而不是留档报告。
- 默认日期：不显示日期、时间、当前日期、导出时间或生成时间。
- 多页输出：必须使用同一套钴蓝、同一类标题倾斜、同一类 UI 标记、同一类颗粒/半调质感；通过构图和页面角色变化，而不是更换风格。

## 排版规则

- 主标题：超大、斜体或倾斜、近黑，允许跨越多个区域。
- 副标题：小到中号，细字重，提供一句上下文。
- 信息点：短句化，常规无衬线，最多 2-3 个。
- 英文标签：只作为 metadata，使用极小字号，例如 `REC`、`//02`、`FIELD NOTE`。
- 竖排文字：可放在右侧或左侧边缘，作为网址、章节、主题标签或项目短码。
- 蓝色信息条：直角横条或短条，承载 1 句短文案或强调关键词。
- 图形符号：蓝色星号/爆炸形/几何符号可作为视觉锚点，但不要每页都放在同一位置。
- 对齐：左对齐为主，可使用大标题与底部摄影错位叠压；不要居中堆叠成普通模板页。

## 信息密度

默认低密度，强调演示现场的冲击力和可读性：

- 1 个主标题
- 1 个短副标题或核心结论
- 2 到 3 个短信息点
- 1 个视觉锚点
- 1 到 3 个极小 UI 标记

如果用户要求报告型页面，可增加为 3 到 4 个模块，但仍必须保持大留白、短句、蓝色克制和清晰层级。

## 时间与日期规则

默认不放任何日期、时间、时间戳、导出时间、生成时间或“今天”。如果源内容本身是时间线、日报、周报、发布会、历史事件或用户明确要求显示日期，必须先确认准确日期再写入提示词或画面。

日期格式优先：

- 中文内容：`YYYY年M月D日`
- 技术或日志语境：`YYYY-MM-DD`
- 英文 UI metadata：仅在用户要求英文风格时使用短英文日期

## 负面约束

- 不要复刻参考图中的招聘内容、岗位名称、网站、工作室标识或具体人物语义。
- 不要把每页都做成招聘海报或单张广告海报；普通内页要服务 PPT 信息表达。
- 不要使用圆角卡片、软阴影、玻璃拟态、渐变光效、霓虹大屏、暗色 HUD。
- 不要使用彩色插画、3D 图标、卡通人物、贴纸风元素。
- 不要让蓝色变成紫蓝渐变或多色调色板。
- 不要添加黑色外框、相机取景框边角或完整矩形画布边框。
- 不要生成不可读中文、乱码英文、假 logo、水印或跨页不一致的品牌字标。
- 不要在没有确认日期时写入日期或时间。

## 质量检查

- 远看 1 秒能否读到主标题。
- 是否只使用一个高饱和钴蓝强调系统。
- 是否保留足够冷白留白，而不是填满内容。
- 黑白摄影是否服务内容，而不是抢掉标题。
- 内页是否不像封面海报那样过度视觉化。
- 直角、硬边、蓝条、REC/UI 标记是否跨页统一。
- 是否没有全页黑色外框或贴边描边。
- 是否没有把参考图里的招聘语义带入新主题。
- 中文标题和短句是否清晰可读。
- 是否没有未确认日期、随机英文假字或模型生成 logo。

## Reuse Prompt Prefix

Apply the `rec-blue-editorial-deck` Style Lock. Create an Image 2-only full-slide PPT image with a cold white editorial deck style: black-and-white grainy photography, high-saturation cobalt blue as the only accent, oversized italic sans-serif headline, hard-edged rectangular layout, generous negative space, subtle print noise and halftone texture, tiny REC/interface metadata, vertical labels, and restrained Chinese-first slide text. Keep the same cobalt blue, typography mood, UI marker system, hard-edge geometry, low text density, and no-logo/no-date defaults across the whole deck. Do not add any black border or full-page outline on the outer canvas edge, and do not copy the recruitment content from the reference image.
