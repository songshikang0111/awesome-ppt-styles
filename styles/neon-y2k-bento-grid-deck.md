# neon-y2k-bento-grid-deck

别名：霓虹 Y2K Bento Grid 风、酸绿黑底拼贴演示风、Dirty Pop Bento Deck、实验贴纸网格 PPT 风格。

## 风格定位

生成具有黑底霓虹、Y2K 互联网拼贴、bento grid 模块、实验字体混排、像素贴纸和潮流视觉模板气质的图片版 PPT、创意提案页、品牌 moodboard、社媒内容策略页、视觉系统展示、作品集页、设计灵感页和青年文化报告页。参考图中的 `BENTO GRID`、Dirtybarn、Grazia、plastic、具体人物、署名和模板文案只作为视觉样本，不作为默认内容；后续使用时应转译为横向 PPT 的封面、目录、观点页、案例页、元素库页和总结页。

核心气质：潮流、实验、碎片化、互联网怀旧、强视觉系统、设计感。适合视觉设计提案、品牌方向探索、创意资产库、社媒栏目包装、青年趋势、AI 视觉风格盘点和需要显得“很会做图、很会排版”的演示。

## 视觉 DNA

- 近黑背景作为统一舞台，所有内容以大小不一的圆角 bento 卡片散布其上。
- 主色是酸橙绿/荧光黄绿，用于大标题、模块边框、标签条、图标和高亮信息。
- 辅色包括电蓝、亮橙、紫色、热粉、白色和少量棋盘格黑白，用于不同模块区分。
- 字体混排明显：超宽未来感标题、像素字、粗黑体、压缩字体、复古 serif、手写字和描边字可以共存，但必须由网格统一。
- 图像语言混合照片、抠图人物、3D 软胶物体、像素纹理、贴纸图标、花朵、笑脸、箭头、网格线、星芒、蝴蝶、火焰、棋盘格和低保真 UI。
- Bento 卡片尺寸差异大：横条、竖卡、方块、小徽章、照片块、文字块和图标块组合成强节奏。
- 画面像视觉资产拼盘或设计系统样张，但转成 PPT 时每页必须有明确主题和阅读路径。

## Style Lock

用于多页图片版 PPT 的跨页控制系统：

- 字体气质：主标题使用超宽、未来感、Y2K、运动科技感无衬线，接近 Eurostile Extended、Bank Gothic、Orbitron、Microgramma、Druk Wide、Arial Black Extended；中文标题应使用宽体黑体、几何黑体或强烈压扁/拉宽的标题黑体。模块内允许混用像素字体、粗黑体、复古 serif、手写字和描边字，但每页必须有一个主标题系统。避免优雅商务衬线、咨询报告字体和普通企业模板字体。
- 语言规则：默认中文优先，尤其标题、章节名、模块名、结论句和输出文档正文必须优先中文；英文只保留产品名、品牌名、平台名、模型名、代码/API 术语和必要专有名词。英文短词可作为贴纸、像素标签、Y2K metadata 或模块装饰，但不能生成无意义长英文。
- 字号层级：H1 是画面最大文字元素，占画面宽度 40%-75%，通常使用酸橙绿或白色超宽体；H2 约为 H1 的 28%-42%；模块标题约为 H1 的 16%-26%；正文约为 H1 的 9%-15%，必须短句化。每页可以有多个小字块，但必须保持一个主阅读焦点。
- 版式网格：默认 16:9 使用 48-88px 安全边距。封面可使用黑底 + 大酸绿标题 + 分散 bento 模块。内页使用 5-10 个不同尺寸模块，形成明确 bento grid；模块之间留出黑色间距，整体不贴边。允许故意错位和 collage 感，但不得失去网格秩序。
- 页面角色：封面、moodboard、视觉资产页可以更碎片化，展示多模块拼贴。观点页、方法页、案例页必须更清楚，减少模块数量，让 1-2 个大模块承担主题，其他小模块做辅助。不要每页都做成纯素材拼盘。
- 装饰元素：可重复使用酸绿描边、像素条、圆角卡片、贴纸小图标、3D 软胶 blob、棋盘格、网格底纹、火焰/花朵/笑脸/箭头、低保真 UI 框、星芒、弯曲箭头和小号标签。每页选择 4-8 种，跨页保持同一图形语法。
- 风格隔离：只使用黑底霓虹、Y2K bento grid、实验拼贴和贴纸系统，不混入浅蓝 starter pack、温暖创作者照片风、高饱和活动海报、暗黑霓虹营销战役、黑银性能规格、冷白 REC 编辑风、深色 HUD 或咨询报告网格。不要复制参考图里的 `BENTO GRID`、Dirtybarn、Grazia、plastic、具体人物、品牌署名、模板购买文案或原始模块内容。
- 色彩比例：45%-60% 近黑背景，12%-22% 酸橙绿/荧光黄绿，8%-15% 电蓝，5%-12% 亮橙，5%-10% 紫/热粉，5%-12% 白色内容块和文字，少量黑白棋盘格。颜色可以强烈，但必须被黑底和圆角模块统一，不要变成无序彩虹噪声。
- 色号：近黑 `#050505` / `#090909` / `#111111`，酸橙绿 `#B8FF1A` / `#A6FF00` / `#C6FF00`，电蓝 `#0B68FF` / `#135CFF`，亮橙 `#FF4B12` / `#FF5A1F`，电紫 `#7A2BFF` / `#8C35FF`，热粉 `#FF3BA7`，标题白 `#F6F2E8`，卡片奶白 `#EFE9DD`，边框绿 `#C6FF00`，弱文字 `#A7A7A0`。
- 边框与容器：统一使用圆角 bento 卡片、酸绿细描边、像素边框、黑底模块间距、贴纸式小圆角标签和低保真 UI 框。圆角可在 8-22px，卡片可以有 1-2px 亮色描边或轻微内阴影。禁止玻璃拟态、柔和商务阴影、全页外框和传统卡片式 SaaS dashboard。
- 页眉页脚行为：可使用小号模板编号、章节 tag、像素 metadata、坐标、模块名或设计系统短码。不要默认添加日期、时间、生成时间、导出时间或当前日期。不要生成真实作者署名、品牌水印、购买提示或 `.AI editable` 这类模板销售文案。
- Logo/字标行为：默认不让 Image2 生成 logo、品牌字标、水印、署名或版权信息。需要品牌时预留角落或底部位置，后期作为固定资产叠加；不要要求模型跨页复刻同一个 logo。
- 文本密度：默认中等密度。封面 1 个巨大标题 + 1 个短副标题 + 4-8 个视觉模块。普通内页 1 个标题 + 1 个短结论 + 4-7 个模块，其中大模块 1-2 个，小模块只放短标签或图标。报告型内容要压缩成关键词、短句、数字和视觉卡片。
- 禁用项：不要复制原始模板文案和署名，不要生成品牌水印，不要纯素材拼盘没有主题，不要密集不可读小字，不要传统商务 bento dashboard，不要浅色温柔手帐风，不要未确认日期。

## PPT 页面适配

- 封面页：黑底 + 巨大酸绿宽体标题 + 6-10 个 bento 模块组成视觉拼盘。
- 目录页：3-5 个章节模块，每个模块用不同颜色和贴纸图形表示。
- 观点页：1 个大标题模块 + 3-5 个证据/关键词小模块。
- Moodboard 页：照片、色块、字体样张、图标和关键词形成视觉资产墙。
- 案例页：一个大案例图 + 多个小标签、指标、图标和点评卡片。
- 框架页：中心大模块 + 周围小模块，展示系统构成。
- 清单页：多个小圆角卡片，每张只放 1 个短句或图标。
- 对比页：左右两组 bento 模块，使用酸绿箭头或像素标签连接。
- 结论页：巨大短句 + 少量模块散落，保留黑底呼吸感。
- 结束页：一个醒目关键词 + 3 个下一步小模块，不使用传统商务 Thank You 模板。

PPTX 交付时默认把整页图片铺满幻灯片，不要求文字、数字、图表、标签可编辑。优先用 Image2 直接生成完整页面视觉。

## 图片构图规则

1. 先确定一个主标题或核心概念，再围绕它分配 bento 模块。
2. 每页需要 1-2 个大模块作为阅读入口，其余小模块只能做辅助。
3. 模块之间必须有稳定黑色间距，不能贴成无边拼图。
4. 酸绿标题或边框应统一全页，不要每个模块都换主色。
5. 图像、3D 小物、贴纸、像素图标和文字块可以混合，但每个模块只表达一个意思。
6. 允许使用实验字体混排，但正文和关键中文必须可读。
7. 内页不要只是展示素材；必须有观点、结构、案例或行动信息。
8. 不要复制参考图的具体品牌、文案、署名或模板销售信息。

## 默认输出参数

- 默认比例：16:9。
- 默认质量：Image2，高分辨率，边缘锐利，中文清晰，卡片网格干净，贴纸和像素细节清楚。
- 默认输出：图片序列；用户说 PPT 时组装为图片版 PPTX。
- 默认语言：中文优先，英文只作为短标签、设计术语或必要专有名词。
- 默认密度：中等信息密度，适合视觉提案和 moodboard 型演示。
- 默认日期：不显示日期、时间、当前日期、导出时间或生成时间。
- 多页输出：保持同一套黑底、酸绿宽体标题、bento 卡片、Y2K 贴纸、像素元素、3D 小物、混合字体和实验拼贴节奏；通过模块数量、主色侧重和页面角色变化。

## 排版规则

- 主标题：超宽、未来感、酸绿或白色，字距略紧或拉宽均可，但必须有重量。
- 副标题：小号白色或弱灰，作为一句上下文。
- 模块标题：可使用像素字、粗黑体、复古 serif 或手写字，但同一模块内层级清楚。
- 正文：短句，不超过 1-2 行。
- 标签：小号圆角、像素边框或酸绿描边，短词优先。
- 图像模块：照片、抠图、3D blob 或图标必须有硬边/圆角容器。
- 装饰文字：可以作为 texture，但不能成为不可读假英文段落。
- 对齐：整体是 bento grid，不是自由散乱 collage；卡片边缘和间距必须有视觉秩序。

## 信息密度

默认中等密度，强调视觉系统和可扫读：

- 1 个巨大标题或核心概念
- 1 个短副标题或核心结论
- 4 到 7 个 bento 模块
- 1 到 2 个大图像/主模块
- 3 到 6 个小贴纸、图标、像素标签或装饰元素

如果用户要求 moodboard 或素材页，可增加到 8-12 个模块，但必须降低文字量，让模块主要承载图像、色块和短标签。

## 时间与日期规则

默认不放任何日期、时间、时间戳、导出时间、生成时间或“今天”。如果源内容本身是趋势报告、版本发布、活动档期、内容日历或用户明确要求显示日期，必须先确认准确日期再写入提示词或画面。不要根据参考图模板编号、署名或水印自动生成日期。

日期格式优先：

- 中文内容：`YYYY年M月D日`
- 设计系统/版本语境：`YYYY-MM-DD`
- Y2K metadata：可用短数字标签，但必须来自用户或源内容

## 质量检查

- 远看 1 秒能否读到主标题或核心概念。
- 黑底和酸绿是否形成统一记忆点。
- Bento 模块是否有明确大小层级和稳定间距。
- 颜色是否被黑底统一，是否没有变成无序彩虹。
- 每个模块是否只表达一个意思，是否没有纯装饰堆砌。
- 页面是否像 PPT 单页，而不是模板素材售卖图。
- 是否没有复制参考图的品牌词、署名、模板文案和原始内容。
- 中文是否清晰可读，英文是否只是必要短标签或术语。
- 是否没有未确认日期、虚假账号信息、假 logo 或过度装饰。
